# Example with multiple Slides

import {Fragment, useState} from 'react'
import { Playground, Props } from 'docz'
import ReactSlidy from '../index.js'
import Number from '../examples/Number'
import '../index.scss'

## multiple slides

Use `numOfSlides` prop to determine the number of slides.

<Playground>
  <ReactSlidy numOfSlides={3}>
    <Number num={1} />
    <Number num={2} />
    <Number num={3} />
    <Number num={4} />
    <Number num={5} />
    <Number num={6} />
    <Number num={7} />
    <Number num={8} />
  </ReactSlidy>
</Playground>

## multiple slides when numOfSlides is greater than length

<Playground>
  <ReactSlidy numOfSlides={5}>
    <Number num={1} />
    <Number num={2} />
    <Number num={3} />
  </ReactSlidy>
</Playground>

## with less slides than numOfSlides

Using `numOfSlides` with sanitize the number of shown items if there's not enough children available to be used. For example, if `numOfSlides` is 5, but you have only one slide, it will show only one using the full width.

<Playground>
  <ReactSlidy numOfSlides={5}>
    <Number num={1} />
  </ReactSlidy>
</Playground>

Another example with two slides as children but using `numOfSlides` with a `3` value. It will sanitize the value in order to use the maximum space possible.

<Playground>
  <ReactSlidy numOfSlides={3}>
    <Number num={1} />
    <Number num={2} />
  </ReactSlidy>
</Playground>

## with less slides than numOfSlides but with sanitize as false

You could, however, disable the previous behaviour by using the prop `sanitize` in order to avoid changing the `numOfSlides` prop on the fly. This mean the space will be divided by the numOfSlides even if there's no slides enough to show.

<Playground>
  <ReactSlidy numOfSlides={3} sanitize={false}>
    <Number num={1} />
    <Number num={2} />
  </ReactSlidy>
</Playground>
